📘 AI 戰艦 UI 功能導覽
這套 UI 的核心好處
把「市場判斷、候選篩選、AI 覆核、決策、掛單、持倉監控」整合在同一個工作流裡。你晚上只要照順序往下操作，就能快速完成隔日掛單前的整理與確認。
STEP 1｜市場戰況 先判斷今天能不能打。先看大盤，再決定今晚火力，避免在不適合的盤勢亂掛單。
STEP 2｜CSV 本地初篩 把 TradingView 候選股快速做本地評分，先把不該打、過熱、風報比差的先排除。
STEP 4｜AI 戰術覆核 AI 只負責幫你檢查候選股的題材、法人、新聞與短線風險，不是重新海選市場。
STEP 4.5｜決策模式 這區是本地模型和 AI 的仲裁區，可以本地優先、AI 優先、交集模式，或自動判斷。
STEP 5｜掛單確認區 睡前最後確認明天要掛的標的、停損、停利和風險，避免來回找資料。
STEP 6｜持倉戰情中心 用來管理已進場股票，支援彈窗輸入與自動預填停損停利，節省大量操作時間。
候選詳卡總覽 把候選股放到下方獨立區塊，主流程不會被大量卡片塞滿，需要細看時再展開。
右上角系統設定 把總資金、單日風險、單檔上限、連敗次數放進彈窗，不占主畫面空間。
整體優點
這套 UI 不是單純看資料，而是把你晚上真正會做的事排成固定流程：先看盤、再篩股、再看 AI、再做決策、再掛單、最後管持倉。這樣能減少漏看、少切頁、少重複計算，也比較不容易因情緒亂下單。

同步範圍
這個版本只同步：STEP 1 市場戰況、STEP 2 CSV 與候選結果、STEP 4 AI 覆核、決策模式。你的持倉、戰報、個人火控仍然只存在自己的瀏覽器，不會互相覆蓋。
已內建預設共用 API 與房間代號。正常情況下不用手動輸入，只有你要切換別的同步空間時才需要修改。
建議架構
HTML 放 GitHub Pages，共用資料放 Apps Script / Google Sheet。大家開同一個 UI 能看到同一份市場、候選與 AI 判斷，但持倉仍然各自獨立。
這版會強制補上預設 API 與房間代號。就算舊的 localStorage 沒有填值，也會自動帶入，不需要手動輸入。
